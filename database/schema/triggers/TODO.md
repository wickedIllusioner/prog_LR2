# TODO: Триггеры для БД

Здесь перечислены все триггеры, которые необходимо реализовать.
Каждый триггер сохраняется в отдельном файле вместе с соответствующей триггерной функцией.

## Формат именования файлов

NN_trigger_краткое_описание.sql

Примеры:
- 01_trigger_set_incident_registration_date.sql
- 02_trigger_check_criticality_level.sql

Где NN — номер триггера от 01 до 10.

## Список триггеров

| № | Имя файла | Описание триггера |
|---|-----------|-------------------|

| 05 | `05_trigger_check_remediation_deadline.sql` | Триггер, контролирующий срок устранения уязвимости и фиксирующий превышение допустимого времени обработки |

## Работа с триггерами

1. Каждый выбирает триггеры из списка
2. Создаёт ветку от актуальной main
3. Добавляет файл(ы) в папку `sql/triggers/`
4. Создаёт Pull Request
5. После слияния — триггер считается выполненным

## Требования к реализации

- Каждый триггер должен иметь соответствующую триггерную функцию
- Триггеры должны быть написаны на PL/pgSQL (PostgreSQL)
- Каждый файл должен содержать комментарий с описанием:
  - на какую таблицу и какое событие (BEFORE/AFTER, INSERT/UPDATE/DELETE)
  - уровень срабатывания (FOR EACH ROW / FOR EACH STATEMENT)
  - логика работы триггера

## Пример структуры файла триггера

```sql
-- =====================================================
-- Триггер: Название триггера
-- Описание: Краткое описание того, что делает триггер
-- Таблица: имя_таблицы
-- Событие: BEFORE/AFTER INSERT/UPDATE/DELETE
-- Уровень: FOR EACH ROW / FOR EACH STATEMENT
-- =====================================================

-- Триггерная функция
CREATE OR REPLACE FUNCTION function_name_for_trigger()
RETURNS TRIGGER AS $$
BEGIN
    -- тело функции
    RETURN NEW; -- или RETURN OLD или RETURN NULL в зависимости от ситуации
END;
$$ LANGUAGE plpgsql;

-- Триггер
CREATE TRIGGER trigger_name
    BEFORE INSERT ON table_name
    FOR EACH ROW
    EXECUTE FUNCTION function_name_for_trigger();
