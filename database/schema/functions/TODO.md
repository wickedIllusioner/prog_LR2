# TODO: Пользовательские функции для БД

Здесь перечислены все функции, которые необходимо реализовать.
Каждая функция сохраняется в отдельном файле.

## Формат именования файлов

NN_function_краткое_описание.sql

Примеры:
- 01_function_avg_reaction_time.sql
- 02_function_total_incidents_count.sql

Где NN — номер функции от 01 до 10.

## Список функций

| № | Имя файла | Описание функции |
|---|-----------|------------------|
| 01 | `01_function_avg_reaction_time.sql` | Функция, вычисляющая среднее время реакции на инциденты за указанный период |
| 02 | `02_function_total_incidents_count.sql` | Функция, определяющая общее количество зарегистрированных инцидентов за выбранный временной интервал |
| 03 | `03_function_vuln_count_by_severity.sql` | Функция, возвращающая количество уязвимостей с заданным уровнем критичности |
| 04 | `04_function_check_threat_level.sql` | Функция, проверяющая наличие информации о степени угрозы для конкретного инцидента |
| 05 | `05_function_most_common_threat_type.sql` | Функция, определяющая наиболее часто встречающийся тип угрозы за указанный период |
| 06 | `06_function_incidents_by_employee.sql` | Функция, вычисляющая количество инцидентов, обработанных конкретным сотрудником |
| 07 | `07_function_assets_with_vuln.sql` | Функция, возвращающая список информационных активов, для которых зарегистрированы уязвимости |
| 08 | `08_function_open_incidents_count.sql` | Функция, определяющая количество открытых инцидентов на текущую дату |
| 09 | `09_function_avg_vuln_per_asset.sql` | Функция, вычисляющая среднее количество уязвимостей, приходящихся на один информационный актив |
| 10 | `10_function_employees_high_criticality_incidents.sql` | Функция, возвращающая список сотрудников, ответственных за обработку инцидентов с высоким уровнем критичности |

## Работа с функциями

1. Каждый выбирает функции из списка
2. Создаёт ветку от актуальной main
3. Добавляет файл(ы) в папку `sql/functions/`
4. Создаёт Pull Request
5. После слияния — функция считается выполненной

## Требования к реализации

- Функции должны быть написаны на PL/pgSQL (PostgreSQL)
- Каждая функция должна содержать комментарий с описанием параметров и возвращаемого значения
- Рекомендуется добавлять пример вызова функции в комментарии

## Пример структуры файла функции

```sql
-- =====================================================
-- Функция: Название функции
-- Описание: Краткое описание того, что делает функция
-- Параметры:
--   param1 (тип) - описание
--   param2 (тип) - описание
-- Возвращает: тип и описание возвращаемого значения
-- Пример: SELECT function_name('2024-01-01', '2024-12-31');
-- =====================================================

CREATE OR REPLACE FUNCTION function_name(...)
RETURNS ... AS $$
BEGIN
    -- тело функции
END;
$$ LANGUAGE plpgsql;
